\subsection*{Về mặt kỹ thuật}

\begin{enumerate}
  \item \textbf{Kiến trúc sạch:} Feature-First Architecture + Repository Pattern + Riverpod cho code có tổ chức, dễ maintain.
  \item \textbf{Đa nền tảng:} Ứng dụng chạy tốt trên Android, iOS và Web từ cùng codebase Flutter.
  \item \textbf{Material Design 3:} Giao diện hiện đại với ColorScheme.fromSeed(), hỗ trợ Light/Dark theme.
  \item \textbf{Bảo mật đa lớp:} Firebase Auth → Firestore Security Rules → sinh trắc học → Secure Storage.
  \item \textbf{Đa ngôn ngữ:} Chuyển đổi tức thì Tiếng Việt $\leftrightarrow$ Tiếng Anh.
  \item \textbf{AI fallback:} Ứng dụng không bao giờ crash dù AI API lỗi nhờ cơ chế fallback.
  \item \textbf{Auto-sync:} Dữ liệu đồng bộ tự động mỗi 8 giây mà không cần realtime listener tốn tài nguyên.
\end{enumerate}

\subsection*{Về mặt sản phẩm}

\begin{itemize}
  \item Source code hoàn chỉnh, public trên GitHub: \url{https://github.com/Haidp2005/G13Money}
  \item Tài liệu kỹ thuật chi tiết: kiến trúc, schema, API, user guide (6 file markdown).
  \item Firestore Security Rules và composite indexes.
  \item SePay webhook handler (Firebase Functions + Cloudflare Workers).
\end{itemize}

\section*{2. Hạn chế}

Mặc dù đạt được các mục tiêu đề ra, ứng dụng vẫn còn một số hạn chế:

\begin{enumerate}
  \item \textbf{Chưa hỗ trợ offline mode:} Ứng dụng yêu cầu kết nối Internet để đọc/ghi dữ liệu. Mặc dù Firestore có offline persistence, nhưng chưa được tối ưu cho trải nghiệm offline hoàn toàn.
  \item \textbf{Chưa có export báo cáo:} Chưa hỗ trợ xuất báo cáo tài chính ra PDF hoặc Excel.
  \item \textbf{Chưa có recurring transactions:} Chưa tự động tạo giao dịch lặp lại (tiền thuê nhà hàng tháng, lương).
  \item \textbf{AI fallback đơn giản:} Phân tích local chỉ xử lý keyword cơ bản, chưa phong phú bằng AI thật.
  \item \textbf{Chưa có unit tests:} Do thời gian hạn chế, chưa viết đầy đủ unit tests và integration tests.
  \item \textbf{SePay tích hợp một chiều:} Chỉ nhận giao dịch vào, chưa hỗ trợ tạo lệnh chuyển tiền.
\end{enumerate}

\section*{3. Hướng phát triển trong tương lai}

\subsection*{Ngắn hạn (1--3 tháng)}

\begin{enumerate}
  \item \textbf{Offline mode với SQLite:} Thêm SQLite cache local, đồng bộ hai chiều với Firestore khi có mạng.
  \item \textbf{Export PDF/Excel:} Xuất báo cáo tài chính hàng tháng ra PDF, chia sẻ qua email hoặc messaging apps.
  \item \textbf{Recurring transactions:} Tạo giao dịch định kỳ tự động (lương, tiền thuê, phí dịch vụ).
  \item \textbf{Unit tests:} Viết test cho core services (AuthService, TransactionsRepository) và widget tests cho UI chính.
\end{enumerate}

\subsection*{Trung hạn (3--6 tháng)}

\begin{enumerate}
  \item \textbf{Tích hợp nhiều ngân hàng:} Mở rộng SePay hoặc tích hợp Open Banking API.
  \item \textbf{Smart categorization:} ML tự động phân loại giao dịch dựa trên nội dung chuyển khoản.
  \item \textbf{Home screen widget:} Widget hiển thị tóm tắt tài chính trên màn hình chính (Android + iOS).
  \item \textbf{Biometric lock per-app:} Yêu cầu xác thực sinh trắc khi mở app (không chỉ khi login).
\end{enumerate}

\subsection*{Dài hạn (6--12 tháng)}

\begin{enumerate}
  \item \textbf{Quản lý tài chính nhóm:} Chia sẻ ngân sách gia đình, nhóm bạn bè.
  \item \textbf{Đầu tư tracking:} Theo dõi cổ phiếu, crypto, quỹ đầu tư.
  \item \textbf{OCR scan hóa đơn:} Tự động nhập giao dịch từ ảnh hóa đơn.
  \item \textbf{Financial goals:} Đặt mục tiêu tài chính (mua nhà, du lịch) với tracking tiến độ.
  \item \textbf{Publish lên Store:} Đưa ứng dụng lên Google Play Store và Apple App Store.
\end{enumerate}

\section*{4. Bài học kinh nghiệm}

Qua quá trình thực hiện đồ án, nhóm rút ra các bài học:

\begin{enumerate}
  \item \textbf{Chọn công nghệ phù hợp:} Flutter + Firebase + Riverpod là combo rất hiệu quả cho dự án vừa và nhỏ, giảm thời gian phát triển đáng kể.
  \item \textbf{Kiến trúc là nền tảng:} Feature-First Architecture giúp mỗi thành viên làm việc độc lập trên feature riêng, giảm conflict khi merge code.
  \item \textbf{AI integration cần fallback:} Không nên phụ thuộc hoàn toàn vào external API; luôn cần cơ chế dự phòng.
  \item \textbf{Security từ đầu:} Firestore Security Rules nên được viết ngay từ đầu, không để đến cuối.
  \item \textbf{Tài liệu song hành:} Viết documentation song song với code giúp dễ maintain và onboard thành viên mới.
\end{enumerate}
